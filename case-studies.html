Case Study #01

FinTech Dashboard Redesign

How we brought a legacy banking app up to WCAG 2.1 AAA compliance while modernizing its dark mode aesthetic.

Client

NovaBank Global

Role

Accessibility & Theme Architecture

Impact

100% WCAG AA Pass Rate

The Challenge

NovaBank's enterprise dashboard was heavily reliant on color to convey critical financial states (e.g., red for loss, green for profit). Furthermore, their "dark mode" was a simple CSS invert that resulted in eye-straining contrast ratios (as low as 2.1:1 in some data tables), making it unusable for users with visual impairments.

Solution Architecture

We completely refactored their CSS variables into a semantic token system. Instead of hardcoding colors, we created contextual tokens like --color-status-error-bg.

/* Refactored Token System */
:root {
  --surface-base: #ffffff;
  --text-critical: #d92d20; /* Ratio: 5.1:1 */
}

[data-theme="dark"] {
  --surface-base: #0f172a;
  --text-critical: #fca5a5; /* Ratio: 7.2:1 */
}

Colorblindness & Semantics

To solve the color-reliance issue, we introduced iconography and pattern fills to financial charts. Positive trends were accompanied by an upward arrow icon aria-hidden="true", and negative trends utilized a textured fill alongside the color red, ensuring Deuteranopia users could distinguish the data points immediately.

"The redesign didn't just prevent compliance lawsuits; our internal customer support tickets regarding 'hard to read charts' dropped by 84% in the first month."
— Sarah Jenkins, VP of Product at NovaBank
